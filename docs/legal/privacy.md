# Privacy Policy — Shellify

**Last updated: May 17, 2026**

---

## 1. Overview

This Privacy Policy explains how Shellify ("the App") handles information when you use it.

Shellify is designed as a **local-first** application. It does not operate a backend, does not create user accounts, and does not collect or transmit personal data to Shellify infrastructure.

---

## 2. No Personal Data Collection

Shellify does not:

- collect personal data,
- create or require user accounts,
- track users or their behavior,
- use analytics or crash-reporting SDKs,
- use advertising networks,
- or perform behavioral or demographic profiling.

No user identifier is created, stored, or transmitted by the App.

---

## 3. Data Stored Locally on Your Device

All data generated by your use of Shellify is stored exclusively on your device, in the App's private storage area (`/data/data/io.shellify.app/`), which is not accessible to other apps.

This includes:

- **Web app configurations** — name, URL, icon, colors, category, and per-app settings (fullscreen, ad-block rules, translation settings, lock type, browser engine, user-agent)
- **Categories** — name, icon, color
- **Settings and preferences** — theme, language, default engine and user-agent
- **Cookies** — encrypted per-app cookie jars (AES-256-GCM)
- **WebView storage** — per-app isolated profile directories containing cache, local storage, and IndexedDB (Android 13+)
- **Downloaded icons** — PWA favicons stored as image files
- **Optional icon packs** — Simple Icons brand logo data downloaded on demand and stored locally for display purposes; these icons are trademarks of their respective owners

The App's database is encrypted at rest using SQLCipher (AES-256). Cryptographic keys are protected by the Android Keystore system, which uses hardware-backed key storage on supported devices.

---

## 4. Network Requests

Shellify only communicates with external servers for specific functional purposes listed below. It never transmits your browsing history or personal data to Shellify-owned servers.

| Purpose | Destination | Data transmitted |
|---|---|---|
| Fetch app icon | Your chosen domain + `google.com/s2/favicons` (fallback only) | Domain name |
| Detect PWA manifest | Your chosen domain | HTTP GET request |
| Download icon library | `cdn.jsdelivr.net` | HTTP GET request |
| Download Gecko engine (optional, user-initiated) | `maven.mozilla.org` | HTTP GET request |
| In-page translation (if enabled) | `translate.googleapis.com` | Selected page text + target language |
| Web content | Any website you choose to open | Governed by that website, not by Shellify |

All network requests use HTTPS. Cleartext HTTP is blocked except to localhost.

---

## 5. Translation

If you enable the translation feature for a web app, text from the page you are viewing may be sent to **Google Translate** (`translate.googleapis.com`).

Translation is performed only on demand. Shellify does not store, log, or forward the text transmitted for translation.

Text is processed by Google LLC under the [Google Privacy Policy](https://policies.google.com/privacy) and [Google Terms of Service](https://policies.google.com/terms).

---

## 6. Backup Feature

Shellify optionally generates encrypted backup files (`.pwab`) stored in a folder you choose on your device.

- Backup files are encrypted using a password you define, via Argon2id key derivation and AES-256-GCM encryption.
- Shellify does not store your backup password and cannot access or recover backup contents.
- Backup files may include cookies and session data for the web apps you have saved.
- Backup files are never transmitted to Shellify servers.
- Scheduled backups (if enabled) run locally via Android WorkManager and write only to the folder you selected.

You are responsible for keeping backup files and passwords secure.

---

## 7. Deep Links and Sharing

Shellify allows you to share web app configurations with others via:

- **QR code** — encodes only the app name and URL
- **Android system share sheet** — sends only the app name and URL
- **Deep links** (`shellify://add?url=...` or `https://shellify.app/add?url=...`) — encodes only the app name and URL in base64

Shared data never includes passwords, cookies, or personal settings. No data is transmitted to Shellify infrastructure in this process.

When the App receives a deep link from an external source, it displays a confirmation dialog showing the destination before any action is taken.

---

## 8. Third-Party Websites

Shellify acts as a viewer (WebView container) for third-party websites you choose to open. We have no control over and are not responsible for:

- the content of those websites,
- the data practices, cookies, or tracking of those websites,
- or their security or availability.

Each website you access is governed by that website's own privacy policy.

---

## 9. Cookies and WebView Storage

Cookies are enabled in WebView to support full PWA functionality. Each web app has its own isolated cookie jar and, on Android 13+, its own isolated WebView profile (preventing cross-site tracking between your apps).

Shellify encrypts cookies before storing them locally. It does not read, analyze, or transmit cookie contents.

---

## 10. App Password and Biometrics

You may optionally set a local password or biometric lock in Shellify. If set:

- the password is stored only as a PBKDF2 hash on your device,
- biometric authentication is processed entirely by Android's BiometricManager — no biometric data is accessed or stored by Shellify,
- and neither the password nor any biometric data is transmitted externally.

---

## 11. Children

Shellify does not knowingly collect data from users of any age. No personal data is collected or processed. Users must comply with applicable age restrictions in their jurisdiction and must not use the App to access content prohibited for their age group.

## 12. Security

Shellify applies modern cryptographic practices throughout:

- Database: SQLCipher (AES-256)
- Cookies at rest: AES-256-GCM
- Backup files: Argon2id + AES-256-GCM
- Cryptographic keys: Android Keystore (hardware-backed where available)

No software system can guarantee absolute security. If you discover a security issue, please report it to [contact@shellify.app](mailto:contact@shellify.app).
